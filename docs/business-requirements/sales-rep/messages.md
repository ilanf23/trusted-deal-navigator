## Purpose

In-app messaging between the team and authenticated clients (via the client portal), plus a feed of AI-generated call-rating notifications that flag conversations worth reviewing. It's the only surface where reps see direct messages from clients who are actively logged into their portal.

## Primary user

Sales rep monitoring client conversations + AI quality flags during the workday. Secondary: founders auditing call quality via the rating feed.

## Entry points

- Sidebar nav: **Communication → Messages**
- Notification badge in the top bar (unread count)

## What the user can do here

- See all conversations with clients (one thread per client)
- Open a conversation → read the message history
- Send a new message to the client (delivered in their portal)
- Browse a feed of AI call ratings with reasoning + transcript preview
- Click a call rating to jump to the underlying call

## Key business rules

- Each client gets exactly one conversation thread (`conversations` table; client_user_id is unique)
- Only authenticated client portal users can initiate or reply
- AI call ratings are generated by a background job after a call ends; only ratings below a threshold (or flagged for review) surface here
- Marking a rating as read is per-user; the underlying call record is untouched

## Data shown

| Field | Source | Notes |
|-------|--------|-------|
| Conversation list | `conversations` joined to `users` for client identity | Sorted by most recent message |
| Messages | `messages` filtered by `conversation_id` | |
| Client name / email | `users` joined via `client_user_id` | |
| Call rating | `ai_call_ratings` (or equivalent) | Includes 1–5 score, reasoning, transcript preview |
| Unread state | `read_at` on rating row | |

## User flows

### 1. Reply to a client message
1. Rep clicks a conversation in the list
2. Reads thread → types reply
3. Send → `INSERT INTO messages` → client sees it in their portal next time they load

### 2. Review a flagged call
1. AI rating appears in feed with low score
2. Rep clicks → expands reasoning + transcript preview
3. Clicks through to the call detail on `/admin/calls`
4. `read_at` updated so the row no longer highlights

## Edge cases & known gaps

- No real-time push — reps must refresh / re-enter the page to see new messages
- No typing indicators or read receipts
- Call rating feed and conversations share a page but are conceptually distinct; users sometimes confuse them
- No filter to hide low-priority ratings once seen
