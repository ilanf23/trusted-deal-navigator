## Purpose

The rep's daily email cockpit inside the CRM. Lets a rep read, reply, archive, and send Gmail directly without leaving the platform, and ties every sent message back to a lead for activity history.

## Primary user

Sales rep working inbox-zero style — triaging morning replies, sending follow-ups during outreach blocks, and logging conversations against deals.

## Entry points

- Sidebar nav: **Communication → Gmail**
- Lead detail panel → *Email* tab → *Open in Gmail*
- New-tab return from Google OAuth (`/admin/inbox/callback`)

## What the user can do here

- Connect/disconnect a Gmail account (OAuth)
- Browse threads with label filters (Inbox, Sent, Drafts, custom labels)
- Open a thread and read the full conversation
- Reply, reply-all, or compose new — with signature auto-applied
- Archive, trash, mark read/unread
- Save drafts and pick them back up later
- Hide threads from the CRM view without affecting Gmail
- Send from any connected mailbox if multiple accounts are linked

## Key business rules

- Every outbound send is mirrored into `outbound_emails` for activity feeds and attribution
- Hidden threads are stored per-user in `hidden_email_threads`; they're not deleted from Gmail
- Signature is generated by `getSignatureHtml()` — never hardcoded per user
- OAuth tokens live in `gmail_connections` and refresh via `gmail-auth` edge function
- All Gmail API calls go through edge function proxies (rate-limit + token refresh centralized)

## Data shown

## User flows

### 1. Connect Gmail
1. Rep clicks **Connect Gmail** → redirected via `gmail-auth` edge fn
2. Google OAuth consent screen
3. Callback lands at `/admin/inbox/callback` → stores tokens in `gmail_connections`
4. Mailbox loads

### 2. Reply with signature
1. Rep opens a thread → clicks *Reply*
2. Editor pre-fills with quoted history + signature
3. Send → `gmail-write` edge fn → Gmail API → `outbound_emails` row written

### 3. Hide noisy thread from CRM
1. Rep clicks *Hide* on a thread row
2. `hidden_email_threads` row inserted
3. Thread vanishes from CRM views but remains in Gmail

## Edge cases & known gaps

- Token refresh failures surface as a banner; user must reconnect
- Very long threads (50+ messages) can be slow to render
- Multi-account support exists but UI for switching is buried
- No bulk operations (multi-select archive/label)

### Edge functions
- `gmail-auth` — OAuth + connection status
- `gmail-mailbox` — list/get/labels (read)
- `gmail-write` — send/archive/trash/draft/mark-read

### Permissions
- Route gate: `AdminRoute`
- Only the connecting user can use their own Gmail tokens
